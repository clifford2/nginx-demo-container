Demo Container Error

If you see this page, the nginx demo container has successfully started, but is not working as designed. Further configuration is required.

The most likely cause of this is if the /usr/share/nginx/html directory is not writable. That can happen if the container's root filesystem is mounted read only, and no volume is mounted for /usr/share/nginx/html which is writable by UID {NGINX_UID}.

Look for Kubernetes readOnlyRootFilesystem or podman / docker `--read-only` configurations.

Please see github.com/clifford2/nginx-demo-container/tree/main/deploy for example Kubernetes deployment manifests that addresses this.
